# Next Step (Step 5): Production Readiness Before OpenTelemetry

Before adding Azure Monitor, I'd like to make this deployment a bit more production-like by introducing:

    Readiness and liveness probes (using your /health endpoint)
    Resource requests and limits
    Environment variables via ConfigMap
    Kubernetes Secrets (which we'll later use for the Application Insights connection string)

These are standard Kubernetes practices and will make the OpenTelemetry integration cleaner. Once those are in place, we'll enable Azure Monitor and you'll be able to see requests flowing into Azure with minimal additional changes.


# Step 5 - Make the Deployment Production Ready

We'll add four things:

Feature	Purpose
Readiness Probe	Tells Kubernetes when the application is ready to receive traffic.
Liveness Probe	Restarts the container if the application becomes unhealthy.
Resource Requests/Limits	Prevents one pod from consuming excessive CPU or memory.
ConfigMap	Stores non-sensitive configuration outside the application.

We'll save Secrets for the Application Insights connection string in the next step.



# What we accomplished

Your application is now much closer to a real production deployment:
